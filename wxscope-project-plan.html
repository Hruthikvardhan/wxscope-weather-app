01 Requirement Gathering DONE
Problem Statement
Users want to check live weather quickly without switching between multiple apps. Existing apps are cluttered with ads and unnecessary features. A clean, fast, single-page weather terminal solves this.
User Analysis — 3 Questions Every Dev Must Answer
QUESTION ANSWER FOR WXSCOPE
Who is the user? Anyone who wants quick weather — students, travellers, professionals
What problem am I solving? Get live weather for any city + auto-detect location in one page
What is the MVP? Search bar + current weather display. Everything else is extra.

02 Design Notes (Paper Sketch → Code) DONE
↓ This is how you document a paper sketch before coding. Draw these boxes on paper, then write notes beside them.
Page Layout (Rough Sketch)
┌─────────────────────────┐ │ LOGO CLOCK │ ← Header ├─────────────────────────┤ │ [ Search Bar ] [SCAN] │ ← Search │ history chips here │ ├─────────────────────────┤ │ City Name │ Temp │ ← Main Grid │ Condition │ °C / °F │ (2 columns) ├─────────────────────────┤ │ HUM WIND FEEL VIS │ ← 4 Gauges │ (arc gauges row) │ ├─────────────────────────┤ │ 5-DAY FORECAST TABLE │ ← Forecast └─────────────────────────┘
Color System Notes
Background → #06090f (near black) Panel bg → #0c1220 Text → #ddd0b3 (warm white) WEATHER THEMES: sunny → amber #f5a623 rain → blue #4d9fff storm → orange #ff6644 cloud → grey #8eb0cc snow → ice #aaddff mist → grey #99aaaa One CSS variable --wx-accent controls ALL colors at once
Typography Plan
  • LOGO / big headings → Bebas Neue (tall, bold)
  • Numbers / gauges → Orbitron (sci-fi feel)
  • Body text / labels → Share Tech Mono (terminal)
  • All fonts → Google Fonts CDN
  • Font sizes → clamp() for responsive scaling
  • Letter spacing on labels → 0.3em - 0.4em
Component Decisions
  • Gauges → SVG arc (stroke-dashoffset trick)
  • Background → conic-gradient radar sweep
  • Forecast → HTML table (semantic)
  • Loading → fixed overlay, opacity toggle
  • Error → slide-in banner, auto-dismiss 5s
  • Theme change → JS adds class to body tag
  • Animations → float, pulse-dot, spin — pure CSS
API Endpoints Needed
ENDPOINT 1 — Current Weather /weather?q={city}&units=metric Returns: temp, humidity, wind, condition, icon, country ENDPOINT 2 — 5-Day Forecast /forecast?q={city}&units=metric Returns: list of 40 readings We group by day → 5 days ENDPOINT 3 — By GPS Coords /weather?lat={lat}&lon={lon} /forecast?lat={lat}&lon={lon}
State & Data Plan
  • currentUnit → 'C' or 'F' (in memory)
  • currentWeather → last API response (in memory)
  • currentForecast → last forecast response
  • searchHistory → localStorage array, max 8
  • No global state library needed — just variables
  • On unit toggle → re-render from saved data (no re-fetch)
▸ INTERVIEW ANSWER — "How did you design this project?"
"Before writing any code, I sketched the layout on paper — a two-column main grid for city info and temperature, a gauge row below, and a forecast table. I planned the color system using CSS custom properties so a single variable controls the entire theme. I also mapped out which API endpoints I needed and what data each one returns, so I knew exactly what to render before I touched the code."

05 Deployment — Vercel Step by Step UPCOMING
1
Create a GitHub Repository
Go to github.com → New Repository → name it wxscope-weather → Public → Create
Repository name: wxscope-weather Visibility: Public ✓ (needed for free Vercel)
2
Push Your Code to GitHub
Open terminal in your project folder and run these commands one by one:
git init git add . git commit -m "feat: initial wxscope weather app" git branch -M main git remote add origin https://github.com/YOURUSERNAME/wxscope-weather.git git push -u origin main
3
⚠️ IMPORTANT — Remove API Key Before Pushing
Never commit your real API key to GitHub. Replace it with a placeholder before git push:
// In script.js — replace before pushing: const API_KEY = 'YOUR_API_KEY_HERE'; // You will add the real key in Vercel env variables
4
Import Project on Vercel
Go to vercel.com → Sign in with GitHub → New Project → Import your wxscope-weather repo → Click Deploy
Framework Preset: Other (not Next.js) Root Directory: ./ (leave default) Build Command: (leave empty for static HTML) Output Directory:(leave empty)
5
Add API Key as Environment Variable
In Vercel dashboard → Project Settings → Environment Variables → Add:
Key: OPENWEATHER_API_KEY Value: your_actual_api_key_here ⚠️ Note: For pure HTML/JS projects, env vars don't auto-inject. Best option for student projects: just hardcode key in JS after deploy, or use a simple Node proxy later.
6
Get Your Live URL
After deploy completes, Vercel gives you a live URL automatically. Share this in your portfolio and GitHub README.
Your live URL will look like: https://wxscope-weather.vercel.app Every time you git push → Vercel auto-redeploys ✓
7
Update README.md — This Is What Interviewers Check
Add these sections to your README:
# WXSCOPE — Weather Intelligence Terminal ## 🔗 Live Demo https://wxscope-weather.vercel.app ## 🛠 Tech Stack HTML5 · CSS3 · Vanilla JavaScript · OpenWeather API ## ✨ Features - Real-time weather by city search - Auto GPS location detection - °C / °F toggle (no re-fetch) - 5-day forecast - localStorage search history - Dynamic theme per weather condition - Fully responsive (mobile first) ## 📐 Architecture - Async/Await + Promise.all for parallel API calls - CSS custom properties for dynamic theming - SVG arc gauges with stroke-dashoffset animation - Modular JS — each feature in its own function
▸ INTERVIEW ANSWER — "How did you deploy this?"
"I pushed the project to GitHub and deployed it on Vercel. Every time I push a new commit, Vercel automatically redeploys — that's CI/CD in its simplest form. The live URL is always up to date. I also made sure to not commit the API key to the repository, which is a basic security practice."
